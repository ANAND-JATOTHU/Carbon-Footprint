UI Elements & ComponentsTo achieve the anti-gravity feel, nothing should look anchored to the background.Floating Dashboards: All main content areas (the carbon chart, the action list, the security logs) must sit inside borderless glass cards. Use a subtle inner border (border: 1px solid rgba(255, 255, 255, 0.1)) to define the edges.Drop Shadows (The Levitation Effect): Instead of harsh, tight drop shadows, use wide, highly dispersed colored shadows. When a user hovers over an eco-action card, a faint cyan glow should appear behind it, making it look like it is lifting off the screen.The Security Log Terminal: A dedicated pane on the right side of the screen with a classic terminal UI. Text should fade in from the bottom, logging active rate-limits and valid JWT decodes in real-time.3. Animations & Physics RulesTraditional CSS transitions (e.g., ease-in-out) feel mechanical. Anti-gravity requires spring physics—movements that have a slight bounce and momentum.Entrance Animations: When the dashboard loads, the glass cards should drift upwards into place (translateY from 20px to 0px) while fading in, as if settling into zero-G.Hover States: When a user hovers over a button, it should smoothly scale up (e.g., scale: 1.05) and cast a wider neon shadow.Data Updates: When a user logs an action and their carbon score drops, the number should not just instantly change. It should spin or count down rapidly like an odometer.Anti-Cheat Triggers: If the user clicks the "Simulate Scripted Spam" button, the UI should shake slightly, and the security log terminal should flash Hacker Pink (#FF0055).4. The Developer Tech Stack & Skills RequiredTo execute this specific UI/UX rulebook, you will need to apply the following technical skills:Frontend TechnologiesFramework: React.js or Next.js. You need component-based architecture to manage all the floating panels independently.Styling: Tailwind CSS. Perfect for rapidly applying glassmorphism (e.g., bg-white/5 backdrop-blur-md rounded-2xl border border-white/10).Animation Engine: Framer Motion (for React). This is non-negotiable for the anti-gravity feel. It allows you to use spring physics (type: "spring", stiffness: 100, damping: 10) instead of flat CSS transitions.Data Visualization: Recharts or D3.js. To render the carbon footprint breakdown as a sleek, neon donut chart with a hollow center.Backend & Integration SkillsWebSocket Integration: The skill to connect your React frontend to your FastAPI backend via WebSockets so the security logs stream onto the UI in real-time without refreshing.State Management: Using React Context or Zustand to manage the live carbon score across the entire app instantaneously.
